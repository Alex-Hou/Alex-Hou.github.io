使用 docker-pelican, github.io 编写博客

Thu 28 July 2016
By Alex Hou

本文介绍在Linux下使用docker-pelican快速搭建一个可用的博客编写环境。

1.安装docker, git

sudo apt-get install docker.io 
sudo apt-get install git

安装完毕docker后需要重启才能生效。 git需要配置用户名、邮箱才能工作：

git config --global user.name "yourname"
git config --global user.email "your@email.address"

2.克隆docker-pelican

这里主要参考docker-pelican GitHub主页的介绍。我个人没有fork这个repo，而是直接下载了。

git clone git@github.com:USERNAME/docker-pelican
cd docker-pelican
make docker-build

然后，按照官网的介绍，修改作者、时区、语言、页面链接等等配置。 接下来就可以试着启动了。

make docker-run

默认配置下，访问localhost:8000即可。 启动后，任何对content目录下文件的修改都会实时更新到output目录，刷新网页即可重新加载。 按照以下模板写一个Hello World吧：

Title: 文章标题
Date: 创建日期
Modified: 修改日期
Category: 文章分类，标志本文处于该分类下
Tags: 文章标签，标志本文处于该标签下
Slug: URL中该文章的链接地址
Author: 作者
Summary: 内容提要
# Hello Wold!

保存为website/content/hello-world.md 试完了别忘记关闭：

make docker-kill

3.开通github.io个人页

打开GitHub Pages或GitHub Pages 帮助页，内容已经很详尽了，下拉式向导，比Windows下安装软件还傻瓜化。 开通后clone到website/output目录下

git clone https://github.com/<yourname>/<yourname>.github.io.git website/output/

4.提交与推送

cd website/output
git add ./*
git commit -a -m "Initial Release"
# 此处引号内的内容是对每次commit的描述
git push origin master 
Username for 'https://github.com':
# 输入Github用户名
Password for 'https://<your@email.address>@github.com': 
# 输入Github密码
# 其实这里可以改为使用SSH协议从而实现免密push
# 参考：https://help.github.com/articles/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent/#platform-linux

然后打开你的主页https://<yourname>.github.io

Voilà
